Introduction

AJAX stands for Asynchronous JavaScript and XML, it is a web development technique for creating asynchronou web applications. AJAX allows web pages to be updated asynchronously by exchanging information with a web server without interfering with the display and the behavior of the current page. That means it updates the part of a web page, without refreshing the entire page.

JSON stands for JavaScript Object Notation. It is a syntax for storing and exchanging data. As the JSON format is text only, it can be sent easily to and from a server. Any data which is updated using AJAX can be stored using the JSON format on web server. We can convert any JSON received from the server into JavaScript objects.

Advantages of AJAX and JSON

  • AJAX reduces bandwidth usage and increases speed: With the help of AJAX, we can eliminate network load and bandwidth usage and retrieve only the information that is required to provide you quicker interfaces and better response time. That means there is no requirement of reloading the page again and again. The response time is faster so that can increase speed and performance.
  • AJAX has high Compatibility: AJAX can be compatible with PHP, ASP.NET, J2EE, or any other languages. It supports almost all popular browsers.
  • AJAX has improved User Experience:The advanced user experience provided by AJAX is by far the most beneficial. It allows webpages to update sequentially by trading a limited amount of information with the server. And because of this, it is possible to update any part of a webpage, without reloading the entire page. Ajax increases the browser's performance and thereby providing a better responsive User Experience.
  • JSON is language independent: JSON syntax is exceptionally simple to utilize and can be used for transferring data in applications using any programming language.
  • JSON is server parsing: For the developers, it is important that on the server-side parsing will be fast so that the users can get a quick response of their response so in this situation JSON server-side parsing is a solid point that demonstrates us to utilize the JSON on the server-side.
  • JSON for sharing data: JSON is the best tool for sharing data of any size including audio, video, etc. That is why JSON stores the data in the arrays which makes data transfer easier and also it is preferable to file format for web APIs and for web development.

Disadvantages of AJAX and JSON

  • AJAX is Time Consuming: AJAX can be consume more time in designing and developing.
  • Security is low in AJAX:Security is less as all files are downloaded at client side in AJAX application. Browsers, which do not support javascript, cannot be used in AJAX application.
  • Exception Handling in JSON: JSON has no exception handling during the JSON calls.
  • Vulnerable: JSON response wrapped in a function call is returned when a JSON service returns. Thus, the Web Application can be vulnerable, used on any untrusted browsers or services and can lead to various types of attacks on the application.
  • JSON has limited data types: As everything is string in JSON, so it might be confusing to know data directly from the JSON file.

Usages

  • AJAX is useful to directly go to the login instead of typing user id and password.
  • With help of AJAX we can simply creat XML http request and retrieve data from txt file.
  • JSON is often used to work with AJAX and it works well together to achieve asynchronous loading of stored data.
  • JSON can be used in forming API's, it can also be used to request data from a different domain by the help of a method called JSONP.

Code Examples

AJAX Structure

Ajax and JSON Syntax Ajax has a structure which is very important to include in a js file, usually AJAX provides several methods for accessing remote files using either "GET" or "POST". It can fetch JSON files by defining the url tag where in url you can load the address and also including the file's dataType. It includes success to run function successfully and error if there is any functional error. 

	$.ajax({
            type: "POST",                 //GET or POST
            url: "example.json",         //Address to load 
            dataType: "json",           //html,xml,json,text
            success: parseJSON,        //function if successful
            error: function (e) {}    //if error occurs 
	});
